## Start here

Go over the files in this folder to understand the standards of this repo.

## Core Standards Documentation

1. **development-standards.md** - Core development principles, architectural patterns, and best practices (START HERE)
2. **component-styling-standards.md** - SCSS module guidelines, design tokens, and styling conventions
3. **storybook-standards.md** - Component documentation patterns for Storybook
4. **tailwind-conversion-standards.md** - Guidelines for converting Tailwind to CSS Modules (mostly complete)

## Key Principle

**ALWAYS START SIMPLE** - Begin with the simplest implementation, iterate based on actual needs, not anticipated complexity.
